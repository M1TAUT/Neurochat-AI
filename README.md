# NeuroChat AI

Продвинутый AI ассистент с веб-интерфейсом и REST API. Проект реализован на FastAPI и использует OpenRouter (DeepSeek) для генерации ответов. При отсутствии API‑ключа приложение работает в локальном *демо‑режиме*, возвращая заглушки.

## Краткий обзор
NeuroChat AI — это небольшой бэкенд для чат‑приложения:
- Веб‑интерфейс (Jinja2 шаблон `index.html`) для начала диалога.
- REST API для статуса, health‑check, получения моделей и общения с AI.
- Интеграция с OpenRouter API (использует модель deepseek/deepseek-chat).
- Демо‑режим при отсутствии API‑ключа (OPENROUTER_API_KEY).

## Возможности
- Веб‑страница с формой начала диалога (шаблон: `templates/index.html`).
- Эндпоинты:
  - / — главная страница (HTML).
  - /api/status — быстрый статус подключения к OpenRouter.
  - /api/health — информация о состоянии сервиса.
  - /api/chat — основной POST‑эндпоинт для чата с AI.
  - /api/models — список доступных моделей (фильтрует DeepSeek).
  - /api/test — тестовый эндпоинт.
- Логирование статуса вызовов к OpenRouter и времени отклика.
- Обработка ошибок и тайм‑аута (30 секунд для вызова API).

## Требования / зависимости
Содержимое `requirements.txt`:

## Установка (локально)
1. Клонируйте репозиторий:
   - git clone <ваш-репозиторий>
2. Перейдите в папку проекта:
   - cd <проект>
3. Создайте и активируйте виртуальное окружение:
   - python -m venv venv
   - Windows: venv\Scripts\activate
   - macOS/Linux: source venv/bin/activate
4. Установите зависимости:
   - pip install -r requirements.txt

## Конфигурация
Переменные окружения:
- OPENROUTER_API_KEY — (опционально) API ключ OpenRouter. Если переменная не установлена, приложение автоматически работает в *демо‑режиме*.
- PORT — порт для запуска (по умолчанию 8000).

Демо‑режим:
- Если OPENROUTER_API_KEY не задан, DEMO_MODE включается автоматически — эндпоинт /api/chat возвращает случайные демонстрационные ответы и не делает внешних вызовов.

Важно:
- В коде указаны заголовки и URLы для OpenRouter: https://openrouter.ai/api/v1/chat/completions и https://openrouter.ai/api/v1/models.
- В исходниках используются HTTP‑заголовки вроде `ContentType` и кастомные `XTitle`/`XVersion` — при необходимости проверьте корректность заголовков (обычно правильно `Content-Type`).

## Запуск
Запуск в режиме разработки:
- Убедитесь, что активировали виртуальное окружение и установили зависимости.
- Экспортируйте ключ (если есть):
  - Linux/macOS: export OPENROUTER_API_KEY="ваш_ключ"
  - Windows (PowerShell): $env:OPENROUTER_API_KEY="ваш_ключ"
- Запустите:
  - python main.py
  - или через uvicorn напрямую: uvicorn main:app --host 0.0.0.0 --port 8000 --reload

После запуска в консоли выводятся ссылки:
- http://localhost:8000 — главная страница
- http://localhost:8000/api/status — статус
- http://localhost:8000/docs — автоматическая документация OpenAPI

## Описание эндпоинтов
1. GET /  
   - Возвращает HTML страницу (Jinja2 шаблон `index.html`).
   - Контекст содержит флаг `demo_mode`.

2. GET /api/status  
   - Проверка подключения к OpenRouter (если в DEMO_MODE возвращает статус demo).
   - Возвращает объект со статусом, сообщением, меткой времени и моделью.

3. GET /api/health  
   - Возвращает статус сервиса, timestamp, версию Python, флаг demo_mode и наличие API ключа.

4. POST /api/chat  
   - Тело: JSON { "message": "текст" }.
   - В демо‑режиме: возвращает один из заранее заданных ответов (tokens_used = 0).
   - В реальном режиме: делает POST к OpenRouter, передаёт system/user сообщения и возвращает ответ AI с полем tokens_used и временем отклика.
   - Обработка ошибок: таймаут (504), проблемы сети (503), и внутренние ошибки (500).

5. GET /api/test  
   - Минимальная проверка работоспособности: возвращает success и список эндпоинтов.

6. GET /api/models  
   - Возвращает список моделей из OpenRouter; в демо‑режиме — ошибка, требующая ключа.
   - Фильтрует модели по вхождению "deepseek" в id и возвращает первые 10.

## Логика и ограничения кода (важно знать)
- DEMO_MODE определяется как not OPENROUTER_API_KEY.
- В payload для /api/chat используется модель "deepseek/deepseek-chat" и ряд параметров (max_tokens, temperature, top_p и т.д.).
- Код в main.py содержит некоторые небольшие опечатки/неточности по наименованиям заголовков (например, `ContentType` вместо `Content-Type`) — это может потребовать правки для корректной работы с внешним API.
- В проекте ожидается папка `templates` с `index.html`. В предоставленном вложении есть `index.html` как отдельный файл, но убедитесь, что при разворачивании он лежит в `templates/index.html`.
- Не видно настройки статических файлов (`StaticFiles`) директории в репозитории; если на веб‑странице используются CSS/JS/изображения — добавьте `static/` и подмонтируйте через FastAPI.

## Рекомендации по безопасности
- Не храните OPENROUTER_API_KEY в публичных репозиториях.
- Для продакшена используйте секретное хранилище (Vault, секреты хостинга) или переменные окружения CI/CD.
- Ограничьте CORS (в коде сейчас указаны пустые списки — нужно явно перечислить разрешенные origin для продакшена).

## Деплой
- Можно развернуть на платформах, поддерживающих ASGI (Render, Fly, Heroku, Railway, Vercel — backend, или любой VPS).
- Для Gunicorn/uvicorn production рекомендуется запустить uvicorn с process manager или использовать systemd / docker:
  - Пример Dockerfile (основная идея):
    - FROM python:3.11-slim
    - COPY . /app
    - WORKDIR /app
    - RUN pip install -r requirements.txt
    - ENV PORT=8000
    - CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
- Убедитесь, что заданы все env vars (OPENROUTER_API_KEY, PORT) в конфигурации развертывания.
- Настройте HTTPS/SSL и ограничьте доступ по IP (если нужно).

## Отладка и устранение неполадок
- Если приложение сообщает, что работает в демо‑режиме — проверьте, установлен ли OPENROUTER_API_KEY в окружении процесса.
- При ошибках от OpenRouter проверьте:
  - корректность названий заголовков (`Content-Type: application/json`),
  - доступность сети и разрешённость исходящих соединений,
  - лимиты и права ключа OpenRouter.
- При таймаутах /api/chat: можно увеличить timeout в requests.post (в main.py timeout=30).
- Если шаблон index.html не загружается — убедитесь, что он находится в директории `templates/` и что Jinja2Templates и StaticFiles корректно настроены.
